# OITELUシステム | 修正指示まとめ

---

## 1. 親機 (`app.py`) の修正指示

### 1.1. UDPブロードキャスト機能の実装
- `broadcast_server_info` 関数を実装し、サーバーのローカルIPアドレスとポート番号（5000）を含むJSONメッセージを、UDPブロードキャストで3秒ごとに送信するロジックを追加してください。
- 送信するポートは `12345` とします。
- 送信するたびに、`add_history` 関数を呼び出して「親機ハートビート送信 (IP: [IPアドレス])」というログを記録してください。
- この関数は、サーバー起動時にバックグラウンドスレッドで実行してください。

### 1.2. APIエンドポイント (`/api/record_usage`) の改修
- このエンドポイントが `card_id` に加えて `unit_name` も受け取るように修正してください。
- 処理内容は以下のように変更してください。
    1.  `unit_name` を元に子機の在庫数と利用可能状態(`available`)を確認します。在庫が0以下、または利用不可の場合はエラーを返します。
    2.  `card_id` を元に利用者の残り利用回数を確認します。回数が0以下の場合はエラーを返します。
    3.  両方のチェックを通過した場合、利用者の残り回数と子機の在庫数をそれぞれ1減らします。
    4.  もし子機の在庫が0になった場合は、その子機の `available` を `0` に更新し、その旨をログに記録してください。
    5.  すべてのDB更新処理は、安全なトランザクションとして実行してください。

### 1.3. 管理者機能の強化
- **手動削除機能**:
    - `admin_user_detail` と `admin_unit_detail` ルートを修正し、`POST` リクエストのフォームデータに `action=delete` が含まれている場合、該当のユーザーまたは子機をデータベースから削除する処理を追加してください。
- **新規子機登録**:
    - `/admin/new_unit` というエンドポイントを新設し、GETリクエストで登録フォーム (`admin_new_unit.html`) を表示し、POSTリクエストで新しい子機をデータベースに登録する機能を追加してください。
- **履歴ページのルート追加**:
    - `/admin/history` というエンドポイントを新設し、`admin_history.html` をレンダリングして全履歴を表示する `admin_history` 関数を追加してください。

## 2. 子機 (`unit_client.py`) の修正指示

### 2.1. GUI (`SettingsGUI` クラス) の全面改修
- `unit_client.py` の先頭にある静的な設定変数を廃止し、`config.json` から設定を読み書きする `load_config` と `save_config` 関数を実装してください。
- **Tkinter GUIのレイアウトを以下のように変更してください。**
    1.  **起動/中止ボタン**:
        - 「保存して起動」ボタンを「起動」ボタンに変更してください。
        - 「起動」ボタンを押すと、クライアントのメインロジックがバックグラウンドスレッドで開始され、ボタンのテキストが「中止」に変わるようにしてください。
        - クライアント実行中は、他のすべての設定ウィジェット（入力欄、ボタン）を無効化(`disabled`)してください。
        - 「中止」ボタンを押すと、バックグラウンドスレッドを安全に停止させ、ボタンのテキストを「起動」に戻し、設定ウィジェットを有効化(`normal`)してください。
    2.  **親機スキャン**:
        - 「親機スキャン」ボタンを押した際、新しい選択ウィンドウは表示せず、ネットワーク上で最初に見つかった親機のURLをサーバーURLの入力欄に自動で設定し、メッセージボックスでユーザーに通知するだけにしてください。
    3.  **ハードウェア状態表示**:
        - 「ハードウェア診断」ボタンを「ハードウェア状態」ボタンに変更してください。
        - ボタンを押すと、新しいウィンドウではなく、GUIウィンドウの**下部**に折りたたみ可能なエリアとしてハードウェア（PCA9685、センサー）の状態が表示されるようにしてください。もう一度押すと非表示になるようにしてください。
    4.  **スレッドエラーの修正**:
        - ハードウェア状態を監視するスレッドが直接TkinterのUIを更新すると `RuntimeError` が発生するため、`queue.Queue` を使用してスレッドからメインのGUIスレッドへ安全に状態情報を渡し、UIを更新するロジックに修正してください。

### 2.2. クライアントロジックの修正
- `run_client` 関数が、GUIからスレッドとして起動・停止できるように、`stop_event` (threading.Event) を引数として受け取るようにしてください。
- `run_client` 内の `send_heartbeat` やNFCの待機ループなどの無限ループは、定期的に `stop_event` をチェックし、セットされていればループを抜けて正常に終了するように修正してください。
- `handle_card_touch` 関数を修正し、親機の `/api/record_usage` エンドポイントにリクエストを送る際、`card_id` と一緒に設定ファイルから読み込んだ `UNIT_NAME` を送信するようにしてください。
