# OITELUシステム 構築・操作マニュアル
**対象読者:** 初めてシステムを構築する学生・管理者

---

## 🔰 第1章: 準備するもの

まずは以下のものが揃っているか確認してください。

1.  **親機用PC**: 普通のノートパソコンでOKです。
2.  **子機用Raspberry Pi**: 電源ケーブルとMicroSDカードも忘れずに。
3.  **NFCリーダー**: パソリ (RC-S380) など。
4.  **Wi-Fi環境**: 親機と子機がインターネットに繋がる必要があります。

---

## 🛠️ 第2章: 環境構築手順

### ステップ 1: どこでも繋がるようにする (Tailscale)
学校のWi-Fiと家のWi-Fiなど、違う場所にいても通信できるように「Tailscale」を使います。

1.  **アカウント作成**: [Tailscale公式サイト](https://tailscale.com/) で登録します。
2.  **親機への導入**: PCにアプリをインストールし、ログインして「Connected」にします。
3.  **子機への導入**: Raspberry Piの黒い画面（ターミナル）で以下を入力します。
    `curl -fsSL https://tailscale.com/install.sh | sh`
    `sudo tailscale up`
4.  **重要**: Tailscaleアプリに表示されている **親機のIPアドレス (例: 100.x.y.z)** をメモしてください。

### ステップ 2: 親機（サーバー）を動かす

#### 方法A: Dockerを使う（推奨）
難しい設定は不要です。「Docker」を使って自動で立ち上げます。

1.  配布されたフォルダ `oiteru_250827_restAPI` を開きます。
2.  フォルダ内でターミナルを開き、以下を実行します。
    ```bash
    docker-compose up --build -d
    ```
3.  ブラウザで `http://localhost:5000` を開き、画面が出れば成功です！

**注意**: Dockerコンテナ内ではUSBデバイスは使用できません。親機PCにNFCリーダーを接続する場合は方法Bを使用してください。

#### 方法B: 親機PCにNFCリーダーを接続する場合
親機PCに直接RC-S380などのNFCリーダーを接続して使用する場合：

1.  Pythonパッケージをインストール:
    ```bash
    pip install -r requirements.txt
    ```
2.  NFCリーダー（RC-S380など）をPCのUSBポートに接続
3.  Flaskサーバーを直接起動:
    ```bash
    python app.py
    ```
4.  ブラウザで `http://localhost:5000` を開き、右上に「親機NFCリーダー: 接続中」と表示されればOK

### ステップ 3: 子機（ラズパイ）を動かす

**重要**: NFCカードリーダーは子機（Raspberry Pi）にのみ接続します。親機には接続しません。

1.  フォルダ `oiteru_250827_restAPI` 全体をRaspberry Piにコピーします。
2.  NFCカードリーダー（パソリ）をRaspberry PiのUSBポートに接続します。

**新機能**: 必要な環境（仮想環境とライブラリ）は初回起動時に自動作成されます！手動インストールは不要です。

#### 起動方法A: GUIモード（デスクトップ環境がある場合）
3.  ターミナルでスクリプトを実行します。
    ```bash
    python unit_client.py
    ```
    **注意**: sudo権限は自動的に取得されます。手動で`sudo`を付ける必要はありません。

4.  初回起動時、以下が自動実行されます：
    - `~/.hirameki`仮想環境の作成とライブラリインストール（数分かかります）
    - sudo権限の自動取得

5.  **設定画面** が表示されたら、以下を入力して「起動」を押します。
    * **サーバーURL**: ステップ1でメモしたIPアドレス (`http://100.x.y.z:5000`)
    * **子機名**: 好きな名前（例: `unit-01`）
    * または「サーバー自動検索」ボタンで親機を自動探知

#### 起動方法B: CUIモード（SSH接続やデスクトップなしの場合）【推奨】
3.  CUIモードで起動します:
    ```bash
    python unit_client.py --no-gui
    ```
    **注意**: sudo権限と仮想環境は自動的に処理されます。

4.  初回起動時、選択肢が表示されます:
    ```
    オプションを選択してください:
      1. そのまま起動
      2. 設定メニューを開く
      3. 親機を自動探知して起動
    選択 [1-3]:
    ```

5.  **推奨**: `3`を選択すると親機を自動探知します（Tailscale/ローカルネットワーク対応）

6.  設定メニュー（`2`を選択した場合）では以下を設定できます:
    - サーバーURL、子機名、パスワード
    - モーター種類（SERVO/STEPPER）
    - 制御方法（RASPI_DIRECT/ARDUINO）
    - センサー使用の有無
    - 各種PIN設定
    - ハードウェア診断

7.  次回以降、自動起動する場合:
    ```bash
    python unit_client.py --no-gui --auto
    ```

8.  親機を自動探知して即座に起動:
    ```bash
    python unit_client.py --no-gui --find-server
    ```

## 🎮 第3章: 使い方ガイド

### シナリオA: 新しい人を登録する
1.  親機のブラウザで「学生証を登録する」をクリック。
2.  子機のリーダーにカードをタッチ。
3.  「登録完了」と出ればOK。

### シナリオB: 在庫を補充する
1.  親機のブラウザで `http://localhost:5000/admin` にアクセス。
2.  ログイン後、「在庫管理」メニューへ。
3.  対象の子機の在庫数を入力して更新。

### シナリオC: 備品を受け取る
1.  子機のリーダーに登録済みのカードをタッチ。
2.  「ピッ」と鳴って、モーターが回り、備品が出てきます。
    * ※ 在庫が0の場合や、未登録のカードでは動きません。

## ❓ よくある質問 (FAQ)

**Q. エラーが出て動きません！**
A. まずは「親機」と「子機」がインターネットに繋がっているか確認してください。Tailscaleが「Connected」になっていることが最重要です。

**Q. モーターが回りません。**
A. 子機の設定画面にある「ハードウェア状態」ボタンを押してみてください。配線エラーが出ていれば、ケーブルを挿し直してください。

**Q. カードリーダーが「No such device (Errno 19)」エラーで止まります。**
A. USB接続の不安定さが原因です。以下を試してください:
1. **自動復旧機能**: 最新版の`unit_client.py`は自動的に再接続を試みます（最大5回）
2. **USBケーブルの確認**: USBハブを使わず、直接Raspberry Piに接続してください
3. **電源の確認**: Raspberry Piの電源容量が不足している可能性があります。2.5A以上のアダプタを使用してください
4. **手動でリセット**: それでもダメな場合、以下のコマンドでUSBデバイスをリセットできます:
   ```bash
   sudo systemctl restart udev
   # または
   sudo usbreset 054c:06c3  # PaSoRi RC-S380の場合
   ```
5. **ログの確認**: 親機のダッシュボードで子機のログを確認し、具体的なエラー原因を特定してください
