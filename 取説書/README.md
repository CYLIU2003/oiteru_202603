# 🍬 OITELU (オイテル) システム

**〜 NFCカードで「お菓子」を管理するスマートIoTシステム 〜**

OITELUは、Raspberry PiとNFC技術を活用した、お菓子の在庫管理・提供システムです。
社員証や学生証などのICカードをかざすだけで、自動でお菓子を取り出したり、利用履歴を記録したりすることができます。

**新機能**: 複数の学生証登録機が同じデータベースを共有できるようになりました！

## ✨ 特徴

*   **簡単操作**: ICカードをかざすだけのシンプル操作。
*   **Web管理**: ブラウザから在庫状況や利用履歴をリアルタイムで確認。
*   **どこでも接続**: Tailscale VPNを利用し、離れた場所にある登録機も安全に接続。
*   **Docker対応**: 面倒な環境構築は不要。コマンド一つでサーバーが立ち上がります。
*   **複数登録機対応** ⭐ **NEW**: 複数の学生証登録機が同じMySQLデータベースを共有し、どこからでも同じユーザーデータにアクセス可能。
*   **外部MySQL接続** ⭐ **NEW**: 別マシンのMySQLサーバーに接続可能。

## 🏗️ システム構成図

### 基本構成（シングル登録機）
```
┌─────────────────┐
│  学生証登録機    │  ← 学生証登録・使用状況確認・管理者ページ
│  (親機/サーバー) │
│  + MySQL DB     │  ← データベースも同じマシンに
└────────┬────────┘
         │
    ┌────┴────┐
    │         │
┌───┴───┐ ┌───┴───┐
│ 子機1  │ │ 子機2  │  ← NFCカードリーダー付きRaspberry Pi
└───────┘ └───────┘
```

### 複数登録機構成（推奨）
```
                    ┌─────────────────────┐
                    │   MySQL データベース  │  ← 中央データベース
                    │   (1台で管理)        │
                    └──────────┬──────────┘
                               │
         ┌─────────────────────┼─────────────────────┐
         │                     │                     │
┌────────┴────────┐  ┌────────┴────────┐  ┌────────┴────────┐
│  学生証登録機1   │  │  学生証登録機2   │  │  学生証登録機3   │
│  (1階 受付)      │  │  (2階 ロビー)    │  │  (7号館 1階)    │
│  :5000           │  │  :5001           │  │  :5000           │
└─────────────────┘  └─────────────────┘  └─────────────────┘
        │                    │                    │
   同じマシン上        同じマシン上          別マシン
   
✅ どの登録機からでも：
   - 学生証の新規登録
   - 利用履歴の確認
   - 管理者ページへのアクセス
   - データのエクスポート
```

**メリット**:
- どの登録機で学生証を登録しても、全ての登録機で即座に使える
- 利用履歴は全登録機で共有・確認可能
- 1台のマシンがダウンしても、他の登録機でサービス継続

## 🔰 はじめに（初心者向けガイド）

Dockerやネットワークの専門知識がなくても大丈夫です。このガイド通りに進めれば、約30分でシステムを構築できます。

### 1. 全体の流れ

システムは以下の3種類の機器で構成されます：

| 機器 | 役割 | 説明 |
|:---|:---|:---|
| **学生証登録機（親機）** | 学生証の登録・管理 | PC上でDockerで動作。Web画面から学生証登録・履歴確認・管理者操作を行う |
| **データベースサーバー** | データの保管 | MySQLで全データを一元管理（登録機と同居可） |
| **子機** | カード読み取り | Raspberry Pi + NFCリーダーでカードを読み取り |

**セットアップの流れ:**
1.  **ネットワーク構築**: 各機器が通信できるようにします (**Tailscale**)
2.  **データベース+登録機起動**: Dockerでサーバーを立ち上げます
3.  **子機設定**: Raspberry Piを設定します

---

### 2. ネットワークの準備 (Tailscale)

登録機と子機を安全に接続するために、無料のVPNサービス「Tailscale」を使用します。

#### 2-1. アカウント作成
1.  [Tailscale公式サイト](https://tailscale.com/) にアクセスし、「Get Started」からアカウントを作成します。

#### 2-2. 登録機（PC）へのインストール
1.  登録機となるPCに Tailscale アプリをインストールし、ログインします。
2.  **重要**: Tailscale上のIPアドレス（例: `100.x.y.z`）をメモしてください。これが登録機の住所になります。

#### 2-3. 子機（Raspberry Pi）へのインストール
Raspberry Piのターミナルで以下を実行します。

```bash
# インストール
curl -fsSL https://tailscale.com/install.sh | sh

# 起動とログイン（表示されるURLにPCからアクセスして承認）
sudo tailscale up
```

---

### 3. 学生証登録機（サーバー）のセットアップ

学生証登録機は **Docker** を使って動かします。

#### 3-1. 準備するもの
*   **Docker Desktop**: [公式サイト](https://www.docker.com/products/docker-desktop/)からインストール。
*   このプロジェクトのファイル一式。

#### 3-2. サーバーの起動

**🔹 1台だけで運用する場合（シングル構成）:**
```bash
# MySQL版（推奨）
docker-compose -f docker-compose.mysql.yml up -d

# SQLite版（開発・テスト用）
docker-compose up -d
```

**🔹 複数登録機で運用する場合（推奨）:**

詳細は [🏢 複数登録機構成（新機能）](#-複数登録機構成新機能) を参照してください。

起動したら、ブラウザで `http://localhost:5000` にアクセスしてみましょう。管理画面が表示されれば成功です！🎉

> **Memo**: サーバーを停止したいときは `docker-compose down` と入力します。

---

### 4. 子機（クライアント）のセットアップ

Raspberry Pi（子機）の設定を行います。

#### 4-1. スクリプトのダウンロード
プロジェクトフォルダをRaspberry Piにコピーします。

#### 4-2. 子機の起動（自動セットアップ）
**新機能**: 必要な環境は自動で作成されます！

```bash
# CUIモードで起動（推奨）
python unit_client.py --no-gui

# またはGUIモードで起動
python unit_client.py
```

初回起動時、以下が自動で実行されます：
- ✅ `~/.hirameki`仮想環境の作成
- ✅ 必要なライブラリの自動インストール
- ✅ sudo権限の自動取得

#### 4-3. 登録機の設定
起動後、選択肢が表示されます：
```
1. そのまま起動
2. 設定メニューを開く
3. 親機を自動探知して起動
```

**推奨**: `3` を選択すると、ネットワーク上の登録機を自動で見つけてくれます！

手動設定する場合は、`2`を選択してサーバーURLに登録機のTailscale IPアドレスを入力：
```
サーバーURL: http://100.100.10.10:5000
子機名: unit-01
```

#### 4-4. 完全自動起動
次回以降、設定済みの状態で即座に起動するには：
```bash
python unit_client.py --no-gui --auto
```

## 🏢 複数登録機構成（新機能）

複数の学生証登録機を設置し、**全ての登録機で同じデータベースを共有**できます。
どの登録機で学生証を登録しても、全ての登録機で即座に使用可能になります。

### 概念図

```
                        ┌─────────────────┐
                        │  MySQL データベース │
                        │  (中央で一元管理)  │
                        └────────┬────────┘
                                 │
         ┌───────────────────────┼───────────────────────┐
         │                       │                       │
         ▼                       ▼                       ▼
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│  登録機1号機     │    │  登録機2号機     │    │  登録機3号機     │
│  (1階 受付)      │    │  (2階 ロビー)    │    │  (別建物)        │
│  :5000           │    │  :5001           │    │  :5000           │
└─────────────────┘    └─────────────────┘    └─────────────────┘

🔹 全ての登録機で可能な操作：
   ✅ 学生証の新規登録
   ✅ 利用履歴の閲覧・エクスポート
   ✅ 管理者ページへのアクセス
   ✅ ユーザー情報の編集・削除
```

---

### 構成パターンの選び方

| パターン | 使用場面 | 使用ファイル |
|:---|:---|:---|
| **パターンA** | 同一マシン上で複数の登録機を立ち上げる | `docker-compose.multi-server.yml` |
| **パターンB** | 別々のマシンに登録機を設置する | `docker-compose.external-db.yml` |

---

### パターンA: 同一マシン上で複数登録機

1台のPCで、データベース + 複数の登録機を同時に動かします。

**ステップ1: 起動**
```bash
docker-compose -f docker-compose.multi-server.yml up -d
```

**ステップ2: アクセス**
| 名称 | URL | 用途 |
|:---|:---|:---|
| 登録機1号機 | http://localhost:5000 | 学生証登録・管理 |
| 登録機2号機 | http://localhost:5001 | 学生証登録・管理 |
| phpMyAdmin | http://localhost:8080 | データベース直接操作（上級者向け） |

**ステップ3: 停止**
```bash
docker-compose -f docker-compose.multi-server.yml down
```

---

### パターンB: 別マシンから外部MySQLに接続

別々のPCにそれぞれ登録機を設置し、1台の中央データベースに接続します。

```
┌──────────────────────────────────────┐
│        メインサーバー (PC-A)           │
│  ┌────────────────┐  ┌────────────┐  │
│  │ 登録機1号機     │  │ MySQL DB   │  │
│  │ :5000           │  │ :3306      │  │
│  └────────────────┘  └────────────┘  │
│         IP: 100.114.99.67            │
└──────────────────────────────────────┘
                    │
         ネットワーク（Tailscale推奨）
                    │
┌──────────────────────────────────────┐
│        サブサーバー (PC-B)            │
│  ┌────────────────┐                  │
│  │ 登録機2号機     │←─ PC-AのDBに接続  │
│  │ :5000           │                  │
│  └────────────────┘                  │
└──────────────────────────────────────┘
```

#### 手順

**【メインサーバー側 (PC-A)】**

**ステップ1: データベース付きで起動**
```bash
docker-compose -f docker-compose.multi-server.yml up -d
```

**ステップ2: ファイアウォールでMySQLポートを開く**
```bash
# Linux (Ubuntu/CentOS)
sudo bash scripts/open_mysql_port_linux.sh

# Windows (PowerShell管理者権限)
.\scripts\open_mysql_port_windows.ps1
```

---

**【サブサーバー側 (PC-B)】**

**ステップ1: 接続テスト**
```powershell
# MySQLに接続できるか確認
.\scripts\check_mysql_port.ps1 -TargetHost 100.114.99.67
```

**ステップ2: 設定ファイルを編集**

`docker-compose.external-db.yml` を開き、以下を変更：

```yaml
environment:
  # ★ メインサーバーのIPアドレスに変更
  - MYSQL_HOST=100.114.99.67
  # ★ この登録機の名前と設置場所を変更
  - SERVER_NAME=登録機2号機(7号館)
  - SERVER_LOCATION=7号館1階
```

**ステップ3: 起動**
```bash
docker-compose -f docker-compose.external-db.yml up -d
```

**ステップ4: アクセス**
- ブラウザで `http://localhost:5000` を開く
- メインサーバーと同じユーザーデータが表示されれば成功！

---

### 登録機の追加（3台目以降）

さらに登録機を追加する場合は、パターンBの手順を繰り返してください。
`SERVER_NAME` と `SERVER_LOCATION` を適切な値に変更することで、どの登録機からアクセスしているか区別できます。

### 新機能: データベース抽象化レイヤー

`db_adapter.py`により、SQLiteとMySQLの両方に対応しました。

**環境変数で切り替え:**
```bash
# SQLite使用（デフォルト）
export DB_TYPE=sqlite

# MySQL使用
export DB_TYPE=mysql
export MYSQL_HOST=localhost
export MYSQL_PORT=3306
export MYSQL_DATABASE=oiteru
export MYSQL_USER=oiteru_user
export MYSQL_PASSWORD=oiteru_password_2025
```

### Windows + WSL2 での NFCリーダー利用 (パススルー設定)

Windows上のDockerコンテナからUSBデバイス（NFCリーダー）を直接制御する場合の設定です。

#### 1. ツールインストール
*   Windows側: `winget install --id dorssel.usbipd-win`
*   WSL2側: `sudo apt install -y linux-tools-generic hwdata` 等（詳細は公式ドキュメント参照）

#### 2. デバイスのアタッチ (PowerShell管理者権限)
```powershell
# デバイス一覧確認
usbipd list

# バインド (初回のみ)
usbipd bind --busid <BUSID>

# アタッチ (毎回必要)
usbipd attach --wsl --busid <BUSID>
```

#### 3. コンテナ起動
`docker-compose.yml` には既に `privileged: true` と `/dev/bus/usb` のマウント設定が含まれています。通常通り `docker-compose up -d` で起動してください。

## 📊 データ参照・共有

### 他のPCからデータにアクセスする方法

データベースには個人情報が含まれるため、GitHubには同期しません。以下の方法でデータにアクセスできます。

#### 方法1: 複数登録機構成（推奨）

複数の登録機で同じデータベースを共有し、どこからでもデータにアクセスします。

1. メインサーバーで `docker-compose.multi-server.yml` を起動
2. 各拠点で `docker-compose.external-db.yml` を起動（メインサーバーのIPを指定）
3. 全ての登録機でリアルタイムにデータを共有

詳細は [🏢 複数登録機構成（新機能）](#-複数登録機構成新機能) を参照してください。

#### 方法2: データエクスポート

登録機でExcel形式にエクスポートして共有します。

```bash
# 全データをExcel出力
python data_viewer.py export-all

# 出力されたExcelファイルをOneDrive/Google Driveで共有
```

#### 方法3: Tailscaleリモートアクセス

登録機の管理画面にリモートアクセスします。

1. Tailscaleで同じアカウントにログイン
2. ブラウザで `http://100.x.x.x:5000/admin` にアクセス
3. データをリアルタイムで確認・ダウンロード

## ❓ トラブルシューティング

| 現象 | 確認事項 |
| :--- | :--- |
| **登録機に繋がらない** | Tailscaleが各マシンで「Connected」になっていますか？<br>登録機のファイアウォール設定を確認してください。 |
| **MySQL接続エラー** | `.\scripts\check_mysql_port.ps1 -TargetHost <IP>` で確認<br>ポート3306がファイアウォールで開いていますか？ |
| **別マシンからDBに接続できない** | `.\scripts\open_mysql_port_windows.ps1` を管理者権限で実行 |
| **NFCリーダーが動かない** | USBケーブルはしっかり刺さっていますか？<br>WSL2の場合、`usbipd attach` を忘れていませんか？ |
| **Permission denied** | Raspberry PiでGPIOを操作するには権限が必要です。<br>`sudo python unit_client.py` で実行してみてください。 |
| **データが見られない** | `data_viewer.py`を使用してExcel出力してください。 |

詳しくは [TROUBLESHOOTING.md](TROUBLESHOOTING.md) を参照してください。
