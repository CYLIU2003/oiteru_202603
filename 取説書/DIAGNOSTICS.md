# システム診断機能 (System Diagnostics)

## 概要

OITELU親機に、PC BIOSのようなシステム診断機能を追加しました。起動時に自動で各種チェックを行い、問題がある場合は警告を表示します。

## 実装された機能

### 1. 起動時自動診断

親機（`app.py`）を起動すると、以下の項目を自動的にチェックします：

```
============================================================
                   OITELU システム診断
============================================================

診断開始時刻: 2025-11-27 13:44:29

[  OK  ] Pythonバージョン           Python 3.10.12
[  OK  ] 必須パッケージ             flask, pandas等がインストール済み
[  OK  ] データベース               ユーザー: 10, 子機: 3
[  OK  ] ディスク容量               空き容量: 146.68GB
[WARNING] ネットワーク              ポート5000が既に使用中
[  OK  ] NFCリーダー               親機ではNFCリーダー不要
[  OK  ] 静的ファイル               全ての静的ファイルが存在
[  OK  ] ファイル権限               データベースファイルのアクセス権限正常
[  OK  ] バックアップシステム         最新バックアップ確認
[  OK  ] 設定ファイル               config.json読み込み
[  OK  ] Docker環境               Docker利用可能
```

### 2. 診断項目の詳細

#### ✅ Pythonバージョン
- Python 3.7以上を確認
- 推奨バージョン未満の場合は警告

#### ✅ 必須パッケージ
- Flask、pandas、werkzeugのインストール確認
- 未インストールの場合はエラー

#### ✅ データベース
- データベースファイルの存在確認
- 必須テーブル（users, units, history, info）の存在確認
- ユーザー数と子機数の表示

#### ✅ ディスク容量
- 空き容量を確認
- 500MB未満: エラー
- 1GB未満: 警告
- 1GB以上: 正常

#### ✅ ネットワーク
- ポート5000の利用可能性確認
- 既に使用中の場合は警告

#### ✅ 静的ファイル
- static/、templates/ディレクトリの存在確認
- 重要なファイルの存在確認

#### ✅ ファイル権限
- データベースファイルの読み書き権限確認

#### ✅ バックアップシステム
- バックアップファイルの存在確認
- 最新バックアップの日時表示

#### ✅ 設定ファイル
- config.jsonの読み込み確認

#### ✅ Docker環境
- Dockerのインストール確認（オプション）

### 3. 管理画面からの診断実行

管理画面（`/admin/diagnostics`）から手動で診断を実行できます。

1. 管理画面にログイン
2. 「診断情報」メニューを選択
3. 「🔍 親機診断を実行」ボタンをクリック
4. 診断結果がリアルタイムで表示されます

## ファイル構成

```
diagnostics.py              # 診断モジュール本体
app.py                      # 起動時に診断を実行（修正）
templates/
  admin_diagnostics.html    # 診断結果表示画面（拡張）
```

## 使用方法

### スタンドアロンでの診断実行

```bash
python diagnostics.py
```

診断結果に基づいて起動を続行するか選択できます。

### 親機起動時の自動診断

```bash
python app.py
```

起動時に自動的に診断が実行されます。
致命的なエラーがある場合は、続行するか確認されます。

### APIエンドポイント

```
POST /api/run_diagnostics
```

管理画面から診断を実行する際に使用されます。

レスポンス例:
```json
{
  "success": true,
  "diagnostics": [
    {
      "component": "Pythonバージョン",
      "status": "OK",
      "detail": "Python 3.10.12",
      "timestamp": "2025-11-27T13:44:29.123456"
    },
    ...
  ],
  "timestamp": "2025-11-27 13:44:29"
}
```

## カラー出力

診断結果は色付きで表示されます：

- 🟢 **緑色 (OK)**: 正常
- 🟡 **黄色 (WARNING)**: 警告（動作可能だが要確認）
- 🔴 **赤色 (FAIL)**: エラー（修正が必要）

## トラブルシューティング

### 診断がスキップされる

```
警告: diagnostics.pyが見つかりません。診断をスキップします。
```

→ `diagnostics.py`が同じディレクトリに存在することを確認してください。

### 必須パッケージエラー

```
[ FAIL ] 必須パッケージ    未インストール: flask, pandas, werkzeug
```

→ 以下のコマンドでインストールしてください：
```bash
pip install -r requirements.txt
```

### データベースエラー

```
[ FAIL ] データベース      不足テーブル: users, units
```

→ `app.py`を一度起動してデータベースを初期化してください。

### ポート使用中警告

```
[WARNING] ネットワーク    ポート5000が既に使用中
```

→ 既存のプロセスを終了するか、別のポートを使用してください。

## 設定のカスタマイズ

`diagnostics.py`を編集して、チェック項目を追加・削除できます：

```python
checks = [
    ("Pythonバージョン", check_python_version),
    ("必須パッケージ", check_required_packages),
    # カスタムチェックを追加
    ("カスタムチェック", my_custom_check),
]
```
