🎯 はじめに

OITERUって何？

ICカード（社員証・学生証など）をかざすと、生理用ナプキンが自動で提供されるプライバシー配慮型の配布システムです！

💡 システムの特長
🔒 プライバシー保護 | 📱 簡単操作 | 📊 自動在庫管理 | 🌐 複数拠点対応

🖥️ 親機の起動方法

親機はデータベース（MySQL）を持つメインサーバーです。

方法1: 🚀 ランチャーで起動（一番簡単！）

  1. scripts フォルダを開く
  2. launcher.bat をダブルクリック
  3. 「1. GUI版」 を選択
  4. 起動モード: 「親機」、実行方法: 「Dockerモード」 を選択
  5. 「起動」 ボタンを押す
これだけ！ 🎉

方法2: 📦 Dockerで起動（推奨）

cd C:\Users\RTDS_admin\source\repos\CYLIU2003\oiteru_250827_restAPI
docker-compose -f docker-compose.mysql.yml up -d

確認：docker ps で oiteru_mysql と oiteru_web が Up ならOK

方法3: 🐍 仮想環境で起動（開発向け）

.\venv-start.ps1 parent-mysql
⚠️ MySQLが別途起動している必要があります

方法4: 🔧 通常モードで起動（非推奨・テスト用）

$env:DB_TYPE = 'mysql'
$env:MYSQL_HOST = 'localhost'
$env:MYSQL_PORT = '3306'
$env:MYSQL_DATABASE = 'oiteru'
$env:MYSQL_USER = 'oiteru_user'
$env:MYSQL_PASSWORD = 'oiteru_password_2025'
python server.py

🔗 従親機の起動方法

従親機は、親機のデータベースを参照するサブサーバーです。

方法1: 🚀 ランチャーで起動

  1. scripts/launcher.bat をダブルクリック
  2. GUI版を選択
  3. 起動モード: 「従親機」 を選択
  4. 設定で MySQLホスト を 100.114.99.67 に変更
  5. 「起動」ボタンを押す

方法2: 📦 Dockerで起動

./docker-start.sh external

方法3: 🐍 仮想環境で起動

Windows:

$env:MYSQL_HOST = '100.114.99.67'
.\venv-start.ps1 sub-parent

Linux:

export MYSQL_HOST=100.114.99.67
./venv-start.sh sub-parent

方法4: 🔧 通常モードで起動（非推奨）

$env:DB_TYPE = 'mysql'
$env:MYSQL_HOST = '100.114.99.67'
$env:SERVER_NAME = '従親機A'
python server.py

📡 子機の起動方法

子機はRaspberry Piで動き、NFCカードの読み取りとナプキンの排出を行います。

⚠️ 前準備: config.json を編集してね！
{
  "SERVER_URL": "http://100.114.99.67:5000",
  "UNIT_NAME": "あなたの子機名",
  "UNIT_PASSWORD": "パスワード"
}

方法1: 🚀 クイック起動スクリプト（おすすめ！）

cd /home/pi/oiteru_250827_restAPI/scripts
sudo ./quick_start_unit.sh 100.114.99.67

方法2: 🐍 仮想環境で起動

cd /home/pi/oiteru_250827_restAPI
./venv-start.sh unit

方法3: 🔧 通常モードで起動（非推奨）

cd /home/pi/oiteru_250827_restAPI
sudo python3 unit.py

方法4: 🔄 自動起動の設定

sudo cp oiteru-unit.service /etc/systemd/system/
sudo systemctl enable oiteru-unit
sudo systemctl start oiteru-unit

⚙️ 管理画面の使い方

ブラウザで以下のURLにアクセスしてください：

http://100.114.99.67:5000/admin

🔧 トラブルシューティング

🔌 接続できない

1. Tailscaleは接続してる？

tailscale status

「connected」と出てなければ以下を実行：

sudo tailscale up

2. config.jsonは正しい？

  • http:// で始まる？（https:// じゃない）
  • 最後にスラッシュ / は付いてない？
  • ポート番号 :5000 は付いてる？

📱 NFCカードが読めない

以下のコマンドでデバイスを確認：

lsusb

→ Sony か ACS のデバイスが表示されればOK。
なければUSBを確認してください！

🩹 ナプキンが出ない

  • 在庫は残ってる？（管理画面で確認）
  • 1日の上限に達してない？
  • モーターの配線は正しい？
  • 排出口が詰まってない？

  2. 何が起きたか → 例：「エラーが出た」

  4. 子機の名前 → UNIT_NAME の値

わからないことがあったら、遠慮なく管理者にお問い合わせください！ 😊
