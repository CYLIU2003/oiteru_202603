役割の違い

名前 役割

親機の起動

親機は Windows 側で Docker を使って起動するのがおすすめです。

cd D:\oiteru_202603
docker-compose -f docker-compose.mysql.yml up -d

確認:

docker ps

停止:

cd D:\oiteru_202603
docker-compose -f docker-compose.mysql.yml down

従親機の起動

cd D:\oiteru_202603
.\venv-start.ps1 sub-parent

親機と通信できるネットワークにいることを確認してください。

子機の起動

子機は Raspberry Pi で tmux を使って起動します。

0. Windows PowerShell から SSH 接続する

まず Windows の PowerShell を開いて、Raspberry Pi に SSH 接続します。

ssh pi@<ラズパイのIPアドレス>

例:

ssh pi@192.168.1.50

Tailscale を使っている場合は、Tailscale のアドレスでも接続できます。

ssh pi@100.x.x.x

初回は確認メッセージが出ることがあるので、その場合は yes を入力します。続いて Raspberry Pi のログインパスワードを入力してください。

1. config.json を確認する

{
    "SERVER_URL": "http://100.114.99.67:5000",
    "UNIT_NAME": "2号機",
    "UNIT_PASSWORD": "password123"
}

2. tmux を入れる

sudo apt update
sudo apt install -y tmux

3. tmux を開始する

tmux new -s oiteru

4. プロジェクトへ移動する

cd ~/oiteru_202603

5. 子機を起動する

./venv-start.sh unit

6. tmux から抜ける

Ctrl + B を押してから D を押す

7. 後で戻る

tmux attach -t oiteru

8. 停止する

  1. tmux attach -t oiteru で戻る
  2. Ctrl + C で停止する
  3. 必要なら exit で tmux を閉じる
