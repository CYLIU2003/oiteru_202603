# OITELUシステム 基本設計仕様書
**作成日:** 2025年11月24日  
**バージョン:** 1.0

## 1. システム概要
本システム「OITELU（オイテル）」は、NFCカード（学生証等）認証を用いた物品自動排出システムである。
サーバー（親機）による一元的な在庫・ユーザー管理と、Raspberry Pi（子機）による物理的な排出制御をREST APIによって連携させる。

## 2. システム構成図

[ユーザー]  ---(NFCカード)---> [子機: Raspberry Pi]
                                     |
                                     +---(在庫確認/排出指示)---> [モーター制御]
                                     |
                               (HTTP/JSON)
                                     |
                            [親機: サーバー (PC/Docker)]
                                     |
                                     +--- [SQLite DB] (ユーザー/在庫データ)
                                     +--- [Web管理画面] (ブラウザ)

## 3. ハードウェア仕様

### 3.1 親機 (Server Node)
* **プラットフォーム**: Dockerコンテナが動作するPC (Windows/Mac/Linux)
* **必須リソース**: USBポート (NFCリーダーを親機で使う場合のみ)

### 3.2 子機 (Client Unit)
* **プラットフォーム**: Raspberry Pi (3B/4B/Zero W等)
* **OS**: Raspberry Pi OS (推奨)
* **周辺機器**:
    * **NFCリーダー**: Sony RC-S380 (USB接続)
    * **モーター**: サーボモーター または ステッピングモーター
    * **駆動回路**: PCA9685 (I2C) または ULN2003 (GPIO)
    * **センサー**: 排出検知用フォトインタラプタ (オプション)

## 4. ソフトウェア仕様

### 4.1 技術スタック
| 領域 | 技術要素 | 備考 |
| :--- | :--- | :--- |
| **バックエンド** | Python 3, Flask | REST APIサーバー |
| **データベース** | SQLite3 | 軽量ファイルベースDB |
| **フロントエンド**| HTML5, CSS3, Jinja2 | 管理画面UI |
| **クライアント** | Python 3, nfcpy, RPi.GPIO | 子機制御スクリプト |
| **ネットワーク** | Tailscale | VPNによる拠点間通信 |
| **環境構築** | Docker, Docker Compose | 親機デプロイ |

### 4.2 API エンドポイント一覧 (抜粋)
* `GET /api/health` : サーバー生存確認
* `POST /api/unit/heartbeat` : 子機の生存通知と設定同期
* `POST /api/record_usage` : 利用リクエスト処理 (在庫減算処理含む)
* `POST /api/log` : 子機からのログ収集

## 5. データベース設計 (ER図概略)

* **users (利用者)**
    * `card_id` (PK): 学生証ID
    * `stock`: ユーザーごとの残り回数
* **units (子機)**
    * `name` (PK): 子機識別名
    * `stock`: 子機内の在庫数
    * `password`: 接続認証用
* **history (履歴)**
    * `txt`: 操作ログ・利用ログ

## 6. セキュリティ・運用要件
* **認証**: 子機は `name` と `password` を用いてサーバーと認証を行う。
* **ネットワーク**: Tailscale VPNを使用し、インターネット経由でも安全にイントラネット接続を実現する。
* **可用性**: 子機はサーバー切断時に自律的に再接続を試行する。
