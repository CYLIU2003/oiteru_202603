# OITELU システム 更新履歴 (2026-01-23)

このドキュメントでは、2026年1月23日に実施された機能追加と変更について説明します。

---

## 📋 変更概要

### 1. 管理画面UIのモダン化
### 2. 柔軟な取得上限設定（期間選択機能）
### 3. 子機設定のリモート管理機能

---

## 🎨 1. 管理画面UIのモダン化

### 変更内容

管理画面のデザインを全面的にモダン化しました。

#### 子機一覧 (`/admin/units`)
- **トグルスイッチ**: 利用可能状態をワンクリックで切り替え可能に
- **ステータスバッジ**: オンライン/オフライン状態を視覚的に表示
- **モダンテーブル**: 角丸デザイン、ホバーエフェクト追加

#### 子機詳細 (`/admin/unit_detail/<id>`)
- **カードベースUI**: 情報を見やすく整理
- **子機名は読み取り専用**: システム識別子として使用されるため変更不可に
  - ※ 以前は名前を変更できましたが、通信に使用されるため固定にしました
- **トグルスイッチ**: 利用可能状態の切り替えがAJAXで即時反映

#### 利用者一覧 (`/admin/users`)
- **トグルスイッチ**: 利用許可をワンクリックで切り替え可能に
- **モダンテーブル**: 統一されたデザイン

#### 利用者詳細 (`/admin/user_detail/<id>`)
- **カードベースUI**: 統一されたレイアウト
- **トグルスイッチ**: 利用許可の即時切り替え
- **危険ゾーン**: 削除ボタンを明確に区分

#### 新規子機登録 (`/admin/new_unit`)
- **自動探知セクション**: グラデーション背景で視認性向上
- **トグルスイッチ**: 初期の利用可能状態を設定可能

### 技術的変更

- `style20250506.css` にトグルスイッチ、モダンテーブル、カードスタイルを追加
- 各テンプレートにAJAX処理を追加（ページリロード不要）

## ⏰ 2. 柔軟な取得上限設定

### 概要

以前は「1日あたりの上限」のみでしたが、以下の期間から選択できるようになりました：
- **1日あたり**: 毎日0時にリセット
- **1週間あたり**: 毎週月曜日0時にリセット
- **1ヶ月あたり**: 毎月1日0時にリセット

### 設定画面 (`/admin/settings`)

```
┌─────────────────────────────────────────┐
│  📊 取得上限設定                        │
│                                         │
│  [1日あたり ▼]  [2] 個まで              │
│                                         │
│  📌 現在の設定: 1日あたり 2個まで取得可能│
│                                         │
│  リセットのタイミング:                  │
│  ・1日: 毎日0時にリセット               │
│  ・1週間: 毎週月曜日0時にリセット       │
│  ・1ヶ月: 毎月1日0時にリセット          │
└─────────────────────────────────────────┘
```

### データベース変更

**settingsテーブル**に以下のカラムが追加されました：

| カラム名 | 型 | 説明 |
|---------|---|------|
| `usage_limit` | INTEGER | 期間あたりの取得上限数 |
| `limit_period` | TEXT | 上限期間 (`day`, `week`, `month`) |

※ 既存の `daily_limit` からの自動マイグレーションが行われます

### API変更

#### 設定取得 (`GET /api/settings`)
```json
{
  "auto_register_mode": true,
  "auto_register_stock": 2,
  "usage_limit": 2,
  "limit_period": "day",
  ...
}
```

#### Heartbeatレスポンス
```json
{
  "success": true,
  "stock": 100,
  "usage_limit": 2,
  "limit_period": "day",
  ...
}
```

### 上限超過時のエラー

利用者が期間内の上限に達した場合、以下のレスポンスが返されます：

```json
{
  "error": "Period limit exceeded",
  "message": "1日あたりの取得上限（2個）に達しました",
  "usage_count": 2,
  "usage_limit": 2,
  "period": "day"
}
```

## 🔧 3. 子機設定のリモート管理機能

### 概要

親機の管理画面から子機の設定を確認・変更できるようになりました。

### 子機詳細画面での設定表示

子機がハートビートを送信すると、その設定情報が親機に保存され、管理画面で確認できます。

```
┌─────────────────────────────────────────────────────────┐
│  ⚙️ 子機設定（リモート設定）                            │
├─────────────────────────────────────────────────────────┤
│  ✓ 設定取得済み   最終更新: 2026-01-23 10:30:45        │
│                                                         │
│  🔧 モーター設定              📡 センサー設定           │
│  ├ モータータイプ: [サーボ▼]  ├ 排出検知: [ON]         │
│  ├ 制御方法: [ラズパイ直結▼]  ├ GPIOピン: [22]         │
│  ├ 速度: [50]                 └ タイムアウト: [5]秒    │
│  └ 動作時間: [2.0]秒                                   │
│                                                         │
│  🌐 通信設定                                            │
│  ├ Heartbeat間隔: [30]秒                               │
│  ├ Arduinoポート: [/dev/ttyUSB0]                       │
│  └ PCA9685チャンネル: [15]                             │
│                                                         │
│  [📤 設定を子機に送信]  [🔄 設定を再取得]               │
└─────────────────────────────────────────────────────────┘
```

### 設定同期の仕組み

```
┌─────────────────────────────────────────────────────────┐
│                    設定同期フロー                       │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. 子機が起動し、Heartbeatを送信                      │
│     ↓                                                   │
│  2. Heartbeatに現在の設定情報を含める                  │
│     ↓                                                   │
│  3. 親機がunit_configsに設定を保存                     │
│     ↓                                                   │
│  4. 管理者が管理画面で設定を変更                       │
│     ↓                                                   │
│  5. pending_unit_config_updatesに保留設定を保存        │
│     ↓                                                   │
│  6. 次回Heartbeat時に保留設定を子機に送信              │
│     ↓                                                   │
│  7. 子機が設定を受信し、config.jsonに保存              │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 設定可能な項目

| カテゴリ | 設定項目 | 説明 |
|---------|---------|------|
| **モーター** | MOTOR_TYPE | サーボ / ステッピング |
| | CONTROL_METHOD | ラズパイ直結 / Arduino経由 |
| | MOTOR_SPEED | モーター速度 (0-100) |
| | MOTOR_DURATION | 動作時間（秒） |
| **センサー** | USE_SENSOR | 排出検知センサーの使用有無 |
| | SENSOR_GPIO_PIN | センサーのGPIOピン番号 |
| | SENSOR_TIMEOUT | センサータイムアウト（秒） |
| **通信** | HEARTBEAT_INTERVAL | ハートビート間隔（秒） |
| | ARDUINO_PORT | Arduinoのシリアルポート |
| | PCA9685_CHANNEL | サーボモーター用チャンネル |

### 新規API

#### 子機設定取得 (`GET /api/unit/<unit_name>/config`)
```json
{
  "success": true,
  "unit_name": "unit_001",
  "config": {
    "MOTOR_TYPE": "servo",
    "CONTROL_METHOD": "raspi",
    "USE_SENSOR": true,
    ...
  },
  "last_update": "2026-01-23 10:30:45"
}
```

#### 子機設定更新 (`POST /api/unit/<unit_name>/config`)
```json
// リクエスト
{
  "MOTOR_TYPE": "stepper",
  "MOTOR_SPEED": 80,
  ...
}

// レスポンス
{
  "success": true,
  "message": "設定変更を予約しました。次回ハートビートで子機に同期されます。",
  "pending_config": {...}
}
```

### 子機側の変更 (`archive/unit_client.py`)

- `apply_remote_config()`: 親機からの設定を適用してconfig.jsonに保存
- Heartbeatに現在の設定を含めて送信
- Heartbeatレスポンスから`config_update`を受信して適用

## ⚠️ 注意事項

1. **データベースマイグレーション**: サーバー起動時に自動的に新しいカラムが追加されます
2. **既存設定の移行**: `daily_limit`の値は自動的に`usage_limit`に移行されます
3. **子機の再起動**: 一部の設定変更（モータータイプなど）は子機の再起動が必要な場合があります
